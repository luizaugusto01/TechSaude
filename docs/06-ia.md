# 06 — Módulo de Inteligência Artificial

## Visão Geral

A IA na MedAI Platform não é um recurso isolado — ela está integrada ao fluxo central do sistema. Cada interação com IA é contextualizada com dados reais do paciente e do sistema.

---

## Funcionalidades de IA por Fase

### Fase 1 — Chatbot de Triagem (Paciente)

**Objetivo:** permitir que o paciente descreva sintomas e receba orientação inicial sobre urgência e especialidade.

**Modelo:** OpenAI GPT-4o

**Fluxo:**
```
Paciente descreve sintomas
        ↓
Backend envia mensagem + histórico + dados do paciente para OpenAI
        ↓
IA responde com:
  - pergunta de acompanhamento OU
  - avaliação de urgência (LOW/MEDIUM/HIGH/EMERGENCY)
  - especialidade sugerida
        ↓
Se EMERGENCY → alerta imediato com orientação de pronto-socorro
Se outros → sugestão de agendar consulta
```

**Prompt base (sistema):**
```
Você é um assistente de triagem médica da MedAI Platform.
Seu papel é ajudar pacientes a entenderem seus sintomas e orientá-los 
sobre a urgência do atendimento e a especialidade mais adequada.

Regras:
- NUNCA forneça diagnósticos definitivos
- NUNCA prescreva medicamentos
- Sempre recomende consulta médica para avaliação completa
- Classifique urgência em: LOW, MEDIUM, HIGH, EMERGENCY
- Em EMERGENCY, instrua o paciente a buscar pronto-socorro imediatamente
- Faça perguntas de acompanhamento para entender melhor os sintomas
- Use linguagem simples e empática

Dados do paciente:
- Idade: {age}
- Alergias: {allergies}
- Histórico relevante: {recentConditions}
```

### Fase 2 — Resumo de Histórico (Médico)

**Objetivo:** antes da consulta, gerar um resumo consolidado do histórico do paciente para o médico.

**Modelo:** Claude (Anthropic) — melhor para textos longos e análise clínica

**Entrada:**
- Prontuários anteriores (últimos N registros)
- Exames recentes e resultados
- Medicamentos em uso (receitas ativas)
- Alergias e comorbidades

**Saída:**
```json
{
  "summary": "Paciente do sexo masculino, 45 anos. Acompanhamento de HAS há 3 anos...",
  "keyPoints": [
    "Hipertenso em uso de Atenolol 25mg",
    "Último exame laboratorial há 6 meses: colesterol alterado",
    "Alergia documentada: Penicilina"
  ],
  "alerts": [
    "Exame de colesterol pendente de repetição"
  ]
}
```

**Prompt base:**
```
Você é um assistente clínico especializado em resumir históricos médicos.
Dado o histórico a seguir, gere:
1. Um resumo conciso e clinicamente relevante (máximo 200 palavras)
2. Os pontos-chave mais importantes para a consulta atual
3. Alertas clínicos que o médico deve considerar

Seja objetivo, preciso e use terminologia médica adequada.
Não invente informações — baseie-se apenas nos dados fornecidos.

HISTÓRICO DO PACIENTE:
{patientHistory}
```

---

### Fase 2 — Priorização de Casos (Médico)

**Objetivo:** classificar automaticamente a urgência dos pacientes agendados com base em suas triagens de IA e histórico.

**Lógica:**
- Analisa o nível de urgência da última conversa com o chatbot
- Considera histórico de doenças crônicas
- Prioriza casos `HIGH` e `EMERGENCY` no topo da lista do médico
- Gera badge visual na agenda do médico

## Implementação do AI Service (NestJS)

```typescript
// src/modules/ai/ai.service.ts

@Injectable()
export class AiService {
  constructor(
    private readonly openai: OpenAI,
    private readonly anthropic: Anthropic,
    private readonly conversationRepo: ConversationRepository,
  ) {}

  async chat(patientId: string, message: string, conversationId?: string) {
    const patient = await this.patientRepo.findWithContext(patientId);
    const history = conversationId
      ? await this.conversationRepo.getMessages(conversationId)
      : [];

    const systemPrompt = this.buildTriagePrompt(patient);

    const response = await this.openai.chat.completions.create({
      model: 'gpt-4o',
      messages: [
        { role: 'system', content: systemPrompt },
        ...history,
        { role: 'user', content: message },
      ],
    });

    const reply = response.choices[0].message.content;
    const parsed = this.parseTriageResponse(reply);

    await this.conversationRepo.saveMessage(conversationId, {
      role: 'user', content: message,
    });
    await this.conversationRepo.saveMessage(conversationId, {
      role: 'assistant', content: reply,
    });

    return { reply, ...parsed };
  }

  async summarizeMedicalRecord(recordId: string) {
    const record = await this.recordRepo.findWithHistory(recordId);
    const prompt = this.buildSummaryPrompt(record);

    const response = await this.anthropic.messages.create({
      model: 'claude-opus-4-7',
      max_tokens: 1024,
      messages: [{ role: 'user', content: prompt }],
    });

    return response.content[0].text;
  }
}
```

## Custos e Limites

| Operação | Modelo | Tokens Estimados | Limite por Usuário |
|---|---|---|---|
| Mensagem de chat | GPT-4o | ~500 in / ~300 out | 50 mensagens/dia |
| Resumo de histórico | Claude Opus | ~2000 in / ~500 out | 10 resumos/dia |
| OCR de exame (futuro) | GPT-4 Vision | ~1000 + imagem | 5 exames/dia |

- Rate limiting implementado por `userId` no Redis (ou em memória na fase 1)
- Custos monitorados via logging de tokens consumidos por requisição
- Alertas de custo configurados na conta OpenAI/Anthropic

---

## Segurança e Ética

- A IA nunca fornece diagnóstico definitivo — sempre recomenda avaliação médica
- Dados do paciente são enviados à API apenas com consentimento explícito (aceite nos termos)
- Nenhuma informação identificável é usada para treinar modelos externos
- Respostas de triagem `EMERGENCY` sempre incluem instrução de pronto-socorro
- Logs de conversas são auditáveis pelo médico responsável
